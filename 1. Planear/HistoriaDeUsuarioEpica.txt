Historia de Usuario Epica

1. Nombre del proyecto: Clinica Prado Verde

2. Proceso: Agendamiento de citas medicas

3. Mision(Ahora): La clinica Prado Verde brinda servicios medicos y gestiona las diferentes citas medicas
                  que los pacientes agendan por via telefonica.

4. Vision(Futuro): La clinica Prado Verde seguirá brindando servicios medicos, gestionando las citas agendadas
                   por los pacientes desde el sistema de informacion transaccional, accediendo a las bases de datos
                   del sistema para la correcta gestion de las mismas.

5. Actores: Usuario - Asesor

6. Producto/Servicio: Agendar citas

7. Escenario actual: Los usuarios deben ir personalmente a los hospitales o por via telefónica y hablar con el asesor para agendar las citas
{
        I. El paciente llama a la clinica y pregunta si hay citas disponibles para x consulta

        II. El asesor asiente e informa que hay citas para x dia / 
            El asesor informa que no hay citas disponibles (FINALIZA EL PROCESO)

        III. El paciente acepta / 
             El paciente declina porque no se acomoda a su disponibilidad (FINALIZA EL PROCESO)

        IV. El asesor da el precio y pide confirmación de la cita

        V. El paciente confirma y se genera la cita
}
8. Escenario objetivo: Los usuarios acceden a una plataforma virtual que les permite ver los horarios disponibles para agendar una cita,
                       mostrando tambiern la sede y el medico con el que se va a realizar la consulta.

9. Problematica: Los usuarios y asesores están utilizando más tiempo del necesario, generando que a veces se estanque el sistema e incluso este puede caer
                 y quedar inhabilitado por el exceso de llamadas o personas en la sucursal fisica.

10. Solucion: Un sistema de información transaccional al que se accede por medio de una plataforma virtual,
             que permita ver que horarios se encuentran disponibles en tiempo real para poder agendar una cita.
